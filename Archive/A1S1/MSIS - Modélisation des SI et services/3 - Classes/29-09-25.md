#### Différents modèles pour chaque dimension : 

- Métier/Organisationnel : Business, Processus métier, Rôles, Règles de gestion
- Information : Entité-Association, Relationnel, Objet
- Informatique : Relationnel, Objet

Métier + Informations -> analyse des infos -> modélisation -> mise en œuvre et intégration

classe = véhicule 
véhicule a plusieurs attributs : noVehicle, category, brand, noModel, noMotor, date

Chaque attribut a un **domaine** : Ens. des valeurs possiblement prises par un attribut

Domaines en modé conceptuelle : 
*Mot Texte Ordonné (Unité) Numérique (Unité) entier, entier positive, décimal, réel Booléen Date*

Equivalents en informatique pure : 
*Char Varchar Numeric (Decimal ou Dec) Integer (ou Int ) Float, Real Boolean Date*

Domaines définis comme tel : 

Dom MOIS : Mot {janvier, février, mars, avril, mai, juin, juillet, aout, septembre, octobre, novembre, décembre}

Dom DUREE : Numérique entier (unité seconde)

-----------------------
Attributs peuvent être "multivalués" càd qu'ils ont plusieurs valeurs. On le note Attribut*

Attributs peuvent être "composites" càd composés de plusieurs valeurs indépendantes : 
	ex :  Rue des clous 234, Chloreville les cailloux 02564
	=     Rue des clous|234| Chloreville les cailloux|02564
	<=   Rue                N°    Ville                              CP

Attributs peuvent être "dérivés" càd qu'ils sont calculés à partir d'un autre :
Age/ par ex

Valeur obscure : undefined possible
noVehicle-

Attribut permanent : attribut dont la valeur initiale est définitive : 
catégorie=
