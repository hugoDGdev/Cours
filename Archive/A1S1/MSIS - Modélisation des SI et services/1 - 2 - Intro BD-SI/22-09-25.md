## Bases de la modélisation d'une BD : 

Dans un SGBD, donnée = Nom; Lieu; Valeur
-> Tous liés par un *modèle de données* 

*voir cours p12* -> schéma conceptuel

Schéma externe : Schéma de BD designé pour la vue utilisateur

<u>Deux parties complémentaires à modéliser : </u>
Part technologique (fonctionnalité du SG) 
//
Part de conception (interprétation du monde réel en info.)

-----------------
<u>Modele conceptuel</u> : ens. de concepts et notations pour décrire activités
<u>Modèle conceptuel de données</u> : décrit manière dont sont/devraient être représentées données dans orga, sys info, bd.

------------
#### Concepts généraux de modélisation : 
- Type d'entité/ Classe / Relation : Représentation d'1 chose/ objet / phénomène du monde réel
	- *ex : chauffeur, véhicule, ligne de bus*
- Association : représentation liens de même type entre classes
	- *ex : chauffeur assure ligne ; chauffeur conduit véhicule*
- Attribut : propriété descriptive d'une entité/classe/relation
	- nom, prénom, adresse

---------------
### Abstractions :

Chauffeur -> abstraction de Dupont, Martin... etc.
Véhicule -> abstraction de Bus, Tram, Trolleybus... etc

------------------
### Pertinence : 

Choix important d'uniquement les choses pertinentes pour le sujet à modéliser 

On sélectionne uniquement les concepts pertinents : 
- Pertinent : Chauffeur, Véhicule, Ligne
- Non pertinent : ?

Parmi tous ces concepts, seuls leurs propriétés pertinentes sont sélectionnées : 
- Chauffeur - Nom, Prénom, Adresse, mail, date de naissance
	- Non pertinent : Etat civil, état santé, nombre d'enfants...etc
